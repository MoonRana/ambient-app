### 4. Audio Recording (React Native / Expo)

**Web Implementation (for reference):**
- Uses `navigator.mediaDevices.getUserMedia()`
- `MediaRecorder` API with `audio/webm;codecs=opus`
- Collects chunks every 1 second
- 3-second timeout fallback for `onstop` event

**React Native Equivalent (Expo):**
```javascript
import { Audio } from 'expo-av';

// Request permissions
const { status } = await Audio.requestPermissionsAsync();

// Configure audio mode
await Audio.setAudioModeAsync({
  allowsRecordingIOS: true,
  playsInSilentModeIOS: true,
  staysActiveInBackground: true,
});

// Start recording
const recording = new Audio.Recording();
await recording.prepareToRecordAsync({
  isMeteringEnabled: true,
  android: {
    extension: '.m4a',
    outputFormat: Audio.AndroidOutputFormat.MPEG_4,
    audioEncoder: Audio.AndroidAudioEncoder.AAC,
    sampleRate: 44100,
    numberOfChannels: 1,
    bitRate: 128000,
  },
  ios: {
    extension: '.m4a',
    outputFormat: Audio.IOSOutputFormat.MPEG4AAC,
    audioQuality: Audio.IOSAudioQuality.HIGH,
    sampleRate: 44100,
    numberOfChannels: 1,
    bitRate: 128000,
  },
});
await recording.startAsync();

// Stop and get file URI
await recording.stopAndUnloadAsync();
const uri = recording.getURI();
```

### 8. HealthScribe Polling Logic

```javascript
const POLLING_INTERVAL = 5000; // 5 seconds
const MAX_ATTEMPTS = 60; // 5 minutes total

async function pollHealthScribe(jobName) {
  let attempts = 0;
  
  while (attempts < MAX_ATTEMPTS) {
    const status = await checkJobStatus(jobName);
    
    if (status.status === 'COMPLETED') {
      const results = await fetchResults(status.transcript.uri, status.clinical_data.uri);
      return results;
    }
    
    if (status.status === 'FAILED') {
      throw new Error(status.failure_reason || 'Job failed');
    }
    
    // QUEUED or IN_PROGRESS - continue polling
    await sleep(POLLING_INTERVAL);
    attempts++;
  }
  
  throw new Error('Polling timeout');
}
```

### 9. 4-Step Wizard Flow

```
Step 1: Documents (ID & Insurance)
  - Capture images using expo-camera
  - Call extract-document-info edge function
  - Update session.session_data.patient_info

Step 2: Medications (Pill Bottles)
  - Capture images of medication bottles
  - Call extract-medications edge function
  - Add to session.session_data.medications[]

Step 3: Diagnoses
  - Call infer-diagnoses with medications list
  - Display suggested diagnoses
  - Allow confirm/reject/add manual
  - Update session.session_data.diagnoses[]

Step 4: SOAP Note Generation
  - Stop recording if still active
  - Upload audio to S3
  - Start HealthScribe job
  - Poll for completion
  - Generate SOAP note
  - Sign and finalize
```

### 13. Critical Implementation Notes

1. **Audio Format**: iOS native recording uses `.m4a` (AAC), not `.webm`. The upload function accepts both.

2. **Base64 Size**: Large recordings can produce huge base64 strings. For recordings over 5 minutes, consider chunked uploads or presigned URLs.

3. **Background Recording**: Configure `staysActiveInBackground: true` in audio mode for uninterrupted recording.

4. **Error Handling**: Always provide fallback to ElevenLabs live transcript if HealthScribe fails.

5. **Session Persistence**: Store session ID in AsyncStorage to resume after app restart.

6. **HIPAA Compliance**: All PHI is transmitted over HTTPS. No PHI is logged or stored on device beyond session cache.
